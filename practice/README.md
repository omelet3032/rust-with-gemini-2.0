좋은 생각입니다! `Result<T, E>`와 `match (Ok, Err)` 패턴을 익히는 가장 좋은 방법은 실제 일어날 법한 **실패 시나리오**를 코드로 짜보는 거예요.

점진적으로 난이도를 높여서 **3가지 문제**를 드릴게요. 한 번 도전해 보세요!

### 🟢 문제 1: 간단한 나눗셈 함수 (기초)

숫자 두 개를 받아서 나누는 함수를 작성하세요. 단, **0으로 나누는 경우**에는 에러를 반환해야 합니다.

* **함수명**: `safe_divide`
* **반환 타입**: `Result<f64, String>`
* **조건**:
* 분모가 0이면 `Err("0으로 나눌 수 없습니다".to_string())` 반환
* 성공하면 `Ok(결과값)` 반환


* **연습 포인트**: `main` 함수에서 `match`를 사용해 결과가 `Ok`일 때와 `Err`일 때 각각 다른 메시지를 출력하세요.

---

### 🟡 문제 2: 사용자 나이 검증 (Custom Error 활용)

사용자로부터 문자열로 된 나이를 입력받아 숫자로 변환하는 함수를 작성하세요.

* **준비물**:
```rust
enum AgeError {
    ParseError, // 숫자가 아닌 경우
    InvalidAge, // 0세 미만 또는 150세 초과인 경우
}

```


* **함수명**: `check_age`
* **반환 타입**: `Result<u32, AgeError>`
* **연습 포인트**:
1. `input.trim().parse::<u32>()`를 실행하고 그 결과(`Result`)를 `match`로 먼저 확인하세요.
2. 파싱에 성공하면 나이의 범위를 체크하고, 범위 밖이면 `Err(AgeError::InvalidAge)`를 반환하세요.
3. 모든 조건이 맞으면 `Ok(나이)`를 반환하세요.



---

### 🔴 문제 3: 타이머 설정값 검증 (실전 응용)

사용자가 입력한 "분:초" 형식의 문자열(예: "10:30")을 초 단위 숫자로 변환하는 비동기 함수를 작성하세요.

* **함수명**: `async fn parse_timer_input(input: &str) -> Result<u64, String>`
* **조건**:
1. `:`을 기준으로 문자열을 나눕니다 (`split(':')`).
2. 분이 없거나 초가 없으면 에러를 반환하세요.
3. 분과 초를 각각 숫자로 파싱하고, 실패하면 에러를 반환하세요.


* **연습 포인트**: 중첩된 `match`를 사용하거나, 성공했을 때의 데이터를 `Ok()` 상자에 담아 `return` 하는 흐름을 연습하세요.

---

### 💡 힌트 (포기하고 싶을 때 보세요!)

* 함수 마지막에 `Ok(값)`을 쓸 때는 `return` 키워드를 생략해도 되지만(세미콜론 없이), 함수 중간에서 바로 끝내고 싶을 때는 **`return Ok(값);`** 처럼 명시적으로 써야 합니다.
* `match` 문은 **결과를 반환**할 수 있습니다.
```rust
let val = match result {
    Ok(v) => v,
    Err(_) => return Err(...), // 여기서 함수 자체를 끝내버림
};

```
